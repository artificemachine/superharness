# Identity Core — Project Owner

Project owner. Senior engineer.
Domain and company details are private and should remain outside committed project files.

## Constraints

- Limited weekly bandwidth.
- Small-team or solo execution model.
- Ship > plan. One task per session. No context-switching.

## Anti-Patterns (ranked)

1. Scope creep — starting features before finishing current task
2. Over-planning as procrastination — planning feels productive but isn't shipping
3. Shiny object — new tool/framework before current one is mastered
4. Skipping knowledge capture — loses compound value

## Rules for Every Agent

- Read CLAUDE.md or AGENTS.md before doing anything
- Never edit .env, credentials, tokens
- Never skip security scan in /ship
- Never push directly to main
- Use file system as memory (progress files, plans, checklists)
- Sub-agents return summaries, not raw output
