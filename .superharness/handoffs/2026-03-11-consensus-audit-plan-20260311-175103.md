## Codebase State Assessment

### Task: unify-yaml-parser — EFFECTIVELY COMPLETE

The prior dual-audit remediation (task `claude-review-dual-audit-plan-20260311`) already consolidated
3 duplicate YAML loaders into `engine/yaml_helpers.rb` as a shared Ruby module. Current state:

- **Ruby (production):** Single `yaml_helpers.rb` module with `safe_load()` via `Psych.safe_load()`.
  Used by `contract.rb`, `inbox.rb`, `validate.rb`. Handles Time/Date normalization.
- **Python (test-only):** `yaml` imports exist only in test fixtures (`test_contract_hygiene_ci.py`,
  `test_codex_watcher_pipeline.py`, `test_claude_watcher_pipeline.py`). These are not production parsers.
- **Shell-based parsing:** None found. Scripts invoke Ruby or shell utilities, no grep-based YAML parsing.
- **monitor-ui.py:** Does NOT import yaml. Reads JSON from subprocess calls.

**Recommendation:** Mark `unify-yaml-parser` as **done** in contract. The original "3 parsers" concern
is resolved. Python test imports are appropriate and do not constitute a parser fragmentation risk.

### Task: eliminate-cli-shim-layer — LOW EFFORT, DEAD CODE

The `/cli/` directory contains 15 thin wrapper scripts (5 lines each, ~2KB total). However:

- The main `superharness` dispatcher routes **directly to `/scripts/`** (line 108: `exec_with_optional_project "$ROOT_DIR/scripts/delegate.sh" "$@"`).
- The `/cli/` wrappers are **never called** by the main entry point.
- This is dead code. Removal requires: delete `/cli/`, verify no external symlinks or references.

**Recommendation:** Execute immediately. Lowest effort, highest clarity gain.

### Task: structured-error-logging — MEDIUM EFFORT, HIGHEST IMPACT

39 instances of error suppression remain across 11 shell scripts:

| Script | Instances | Pattern |
|--------|-----------|---------|
| `inbox-watch.sh` | 8 | Lock cleanup, dispatch/deadline calls |
| `inbox-dispatch.sh` | 7 | YAML set operations |
| `reset-watcher-and-test.sh` | 8 | Uninstall, debug output |
| `inbox-deadline-check.sh` | 3 | Ruby contract operations |
| `doctor.sh` | 3 | Diagnostic checks |
| Other scripts | 10 | Mixed patterns |

Exit codes are preserved with `|| true` but stderr context is frequently discarded.
No structured logging infrastructure exists yet.

**Recommendation:** Proceed after cli-shim removal. Define a `log_error()` helper, replace
`2>/dev/null` with captured stderr, replace `|| true` with explicit error handling or logged fallbacks.
