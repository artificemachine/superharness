## Technical Design

### Module Hook
- **on_verify**: Fires `security_scan` action during task verification
- **block_on**: critical — prevents verify success if CRITICAL/HIGH findings detected

### Scanner Detection
- Primary: `shipguard` (preferred)
- Fallback: `gitleaks`
- Graceful skip if neither found

### Blocking Logic
- Non-zero exit code + CRITICAL/HIGH in output → `blocked: true`, `success: false`
- Non-zero exit code without critical severity → `success: true` (warnings don't block)
- Clean scan (exit 0) → `success: true`

### Settings
- `severity_threshold`: "high" (default) — controls blocking threshold
- Timeout: 300s (5 minutes) for long scans
