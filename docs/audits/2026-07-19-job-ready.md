## Verdict: NOT READY

Two independent hard gates trip it:
1. **Personal-data leak** (stage 1): real homelab LAN IPs in HANDOFF.md (<LAN-gpu-endpoint>,
   <LAN-vidistiller-host>) + `$HOME` paths in 8 tracked files, on a public repo.
2. **Unpatched transitive CVEs** (stage 4): `starlette 1.0.0` (multiple 2026 PYSEC,
   DoS-class, fixed 1.0.1/1.3.x) and `python-multipart 0.0.27` (fixed 0.0.30), pulled
   via fastmcp. No `.github/dependabot.yml`, so they drift silently.

No real secrets (gitleaks over 1124 commits: only fake fixtures at
test_module_obsidian.py:211). The engineering underneath is strong — the NOT READY is
about the public-facing last mile and dependency hygiene, not core quality.

- **Undeclared runtime/test dep**: `requests` imported by a test but absent from

  11m14s). README claims "151 tests" — real suite is ~22x larger; claim badly stale.

### Stage 5 — Gauntlet: NEEDS HARDENING
- **Security (SAST)**: shipguard 24x PY-007 (f-string SQL). Assessed LOW real risk —
  interpolated tokens are internal table/column NAMES and version ints; VALUES are
  parameterized (`?`); input path is a local operator CLI, no external attacker.
  Hardening: whitelist column names in inbox_dao.py:358,372. Zero `shell=True` in src.
- **Threat model — dispatch injection: SAFE.** Launchers `exec "$PYTHON3" -m ... "$@"`
  and `exec codex ... "$PROMPT"` pass task text as a single quoted argv element — no
  eval, no `sh -c`. Malicious task text cannot break out to a shell.
- **Senior review: APPROVED-WITH-FIXES.** One MAJOR live bug: lifecycle_rules.py:388
  measures deadline age from `created_at` not `in_progress_at` → an old-but-actively-
  running task can be force-failed on dispatch. Plus 4.5k-line watcher god-module
  (inbox_watch.py) and blanket `except Exception: pass` across ~12 mirror funcs hiding
  watcher drift. Concurrency core (claim_next BEGIN IMMEDIATE, WAL) genuinely sound.
- **QA: READY-WITH-WARNINGS.** Real test pyramid (433 files: ~337 unit, 32 integ, 15
  e2e, chaos, perf). Gap: CI installs pytest-cov but never enforces `--cov-fail-under`.
  Missing concurrent-WRITER chaos test (only readers tested).
- **UX: Adopt.** Strong cold-start onboarding banner; minor verb sprawl across
  task/delegate/dispatch/enqueue/auto-dispatch.
- **Docker: N/A** (no Dockerfile in repo).

### Stage 6 — Architecture: NEEDS HARDENING
- **CRITICAL**: dashboard-ui.py:1624 hardcodes legacy DB path
  (`.superharness/state.sqlite3`) while CLI defaults to XDG
  (`~/.local/state/superharness/<hash>/state.db`) — dashboard and CLI can read two
  different databases for the same project → silent divergence / empty dashboard.
- **HIGH**: no CHECK constraint on tasks.status (state machine enforced only in app
  code, bypassable); 5 pragma-bypassing raw connections (mcp/session.py:57 etc. skip
  FK/busy_timeout/WAL); JSON-in-TEXT columns with no json_valid CHECK and no try/except
  on read → one malformed row breaks `shux contract`.
- MEDIUM: reactive migration-drift healing (hardcoded v25 repair), ON DELETE gaps on
  parent_id / failures / decisions / ledger.
- Positives: real migration framework (schema_migrations + user_version, savepoint-
  wrapped), dispatch idempotency UNIQUE key, watcher singleton CHECK+flock.

### Stage 7 — CI governance: PASS (needs-hardening)
