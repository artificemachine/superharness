## Architecture: Smart Brain

```
Every 5 min (watcher cycle):

  FAILURE → _self_heal()     → pattern match → pip install / kill orphan / pragma
         → _call_fleet()     → Ollama qwen2.5:7b → classify root cause  
         → analyze_failure() → permanent_block? → _maybe_pause_agent()
         → trace.jsonl       → read back next cycle → agent_deprioritized if paused 3+ times

  CONSENSUS → discussions_dao.get_all(status="consensus") → auto-create tasks
```

### Brain layers
1. **Deterministic patterns** → regex match common errors, attempt safe fixes
2. **Ollama (local)** → failure classification (transient/permanent_block/config)
3. **Lab fleet (remote)** → qwopus-27b on VM913 when accessible via SSH tunnel

### Self-healing patterns
| Error pattern | Action |
|--------------|--------|
| `ModuleNotFoundError: No module named 'X'` | `pip install X` |
| `Address already in use` | `kill` orphan on that port |
| `database is locked` | PRAGMA unlock |
| `email.message` / python 3.14 | Suggest downgrade (cannot auto-fix) |
| `pipx/venvs/` path stale | Suggest `pipx install superharness` |

## What Needs Attention

1. **qwen2.5:7b is too slow/unreliable for classification** — 50% accuracy, 18-30s latency. Use deterministic patterns as primary. Lab fleet (qwopus-27b) would be much better but needs SSH tunnel.

2. **Python 3.14 incompatibility** — `email.message` removed from stdlib. Fixed by using pyenv Python 3.11 for launchd operators. Consider downgrading pipx venv Python or patching importlib.

3. **26 plist files still on disk** — Only 2 are loaded (trayzury + watchdog). The other 24 are dormant. They'll auto-heal via watchdog's `--auto-discover` but only when a project's operator dies.

4. **Discussion system needs agent availability pre-check** — Currently starts with `--force`. The availability check (heartbeat) is correct but requires agents to have daemons running.

5. **Test coverage gaps** — Self-healing, circuit breaker, and fleet classification have no integration tests against real Ollama/fleet.

## Decisions Made

1. **Operator operators should use pyenv Python (3.11), not pipx Python (3.14)** — pipx venv paths change on reinstall, breaking KeepAlive; Python 3.14 has stdlib issues.

2. **Fleet config lives in `~/.config/superharness/fleet.yaml`** — user-specific, never committed, survives pipx reinstalls.

3. **Discussion consensus requires ≥2 submissions (for n≥3)** — hardcoded threshold in `_gc_discussion_deadlock` and `_auto_advance_orphaned_rounds`.

4. **Self-healing is pattern-matched, not LLM-generated** — Regex patterns + deterministic actions are safe. LLM-generated fixes would be dangerous.

5. **Only 2 operators loaded (trayzury + watchdog)** — Starting all 25 was the cause of the 403-process leak. Load operators only for active projects.
